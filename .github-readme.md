## ğŸ”Œ API Endpoints

### Profiles
- `GET /api/profiles` - Get all profiles
- `POST /api/profiles` - Create a new profile
- `PATCH /api/profiles/:id/timezone` - Update profile timezone

### Events
- `GET /api/events/profile/:profileId` - Get all events for a profile
- `GET /api/events/:id` - Get a single event
- `POST /api/events` - Create a new event
- `PATCH /api/events/:id` - Update an event
- `DELETE /api/events/:id` - Delete an event

## ğŸ’¡ Usage

### Creating a Profile
1. Click "Select current profile..." in the header
2. Enter a name and click "Add"
3. The profile is created with default timezone

### Creating an Event
1. Select one or more profiles
2. Choose a timezone for the event
3. Set start and end date/time
4. Click "Create Event"

### Viewing Events in Different Timezones
1. Select a profile from the dropdown
2. Use "View in Timezone" dropdown to change display timezone
3. All event times automatically convert

### Updating Events
1. Click on any event card
2. Modify the desired fields
3. Click "Update Event"
4. Changes are logged in update history

## ğŸŒŸ Key Features Explained

### Timezone Handling
Events are stored in UTC in the database and converted to the user's selected timezone for display. When a user changes their timezone, all event times and update logs automatically adjust to show the correct local time.

### Update Tracking
Every event modification is tracked with:
- What changed (field name)
- Previous value
- New value
- Timestamp of change

All timestamps in update logs respect the current viewing timezone.
